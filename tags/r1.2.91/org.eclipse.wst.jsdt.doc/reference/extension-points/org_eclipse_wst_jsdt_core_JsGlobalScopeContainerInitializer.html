Includepath Container Initializers

Identifier:
org.eclipse.wst.jsdt.core.JsGlobalScopeContainerInitializer

Since:
2.0

Description:
This extension point allows clients to contribute custom includepath container initializers, which are used to lazily bind classpath containers to instances of org.eclipse.wst.jsdt.core.IJsGlobalScopeContainer.

Configuration Markup:

<!ELEMENT extension (JsGlobalScopeContainerInitializer*)>

<!ATTLIST extension

point CDATA #REQUIRED

id    CDATA #IMPLIED

name  CDATA #IMPLIED

>

  • point - a fully qualified identifier of the target extension point
  • id - an optional identifier of the extension instance
  • name - an optional name of the extension instance

<!ELEMENT JsGlobalScopeContainerInitializer EMPTY>

<!ATTLIST JsGlobalScopeContainerInitializer

id    CDATA #REQUIRED

class CDATA #REQUIRED

>

  • id - a unique name identifying all containers for which this initializer will be activated.
  • class - the class that implements this container initializer. This class must implement a public subclass of org.eclipse.wst.jsdt.core.JsGlobalScopeContainerInitializer with a public 0-argument constructor.

Examples:
Example of a declaration of a JsGlobalScopeContainerInitializer for a classpath container named "JDK": 
                                                                       
<extension point="org.eclipse.wst.jsdt.core.JsGlobalScopeContainerInitializer">            
   <JsGlobalScopeContainerInitializer                                          
      id="JDK"                                                        
      class="com.example.MyInitializer"/>                           
</extension>
